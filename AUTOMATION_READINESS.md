# AUTOMATION_READINESS

## 루프별 준비도

| 루프 | 자동화 수준 | Codex 역할 | Claude Verifier | 위험도 | 권한 | Rollback | 준비 상태 | 보완점 |
|---|---|---|---|---|---|---|---|---|
| 1. 정적 뼈대 | AUTO | `index.html` / `styles.css` / `script.js` 최소 구조 작성 | 정적 검사, 링크, 반응형 구조, JS 문법 확인 | 낮음 | 파일 편집 | 단일 파일 되돌리기 | 준비됨 | 런타임 브라우저 확인은 별도 |
| 2. 프로필 콘텐츠 | HITL | 확인된 사실만 반영, `[사람 확인 필요]` 유지 | 사실성/누락/톤 점검 | 중간 | 문구 승인 | 섹션 단위 되돌리기 | 부분 준비 | 프로젝트명·성과 최종 승인 필요 |
| 3. Snake 게임 | HITL | 게임 루프, 입력, 충돌, 점수, 재시작 구현 | 조작성, 오류, 반응성 점검 | 높음 | 기능 승인 | 게임 모듈 단위 되돌리기 | 부분 준비 | 입력 방식과 난이도 합의 필요 |
| 4. Retry 수정 | AUTO | 실패 원인 1개와 관련 파일만 최소 수정 | 동일 검증 재실행, fingerprint 비교 | 중간 | 제한적 수정 | 직전 커밋/패치 되돌리기 | 준비됨 | 실패 분류가 명확해야 함 |
| 5. 최종 회귀 | AUTO | 메타, 링크, 접근성, 마감 정리 | 전체 회귀, 배포 전 체크 | 중간 | 마감 승인 | 최종 태그/스냅샷 유지 | 준비됨 | 브라우저 실측 필요 |
| 6. 배포 승인 | HITL | 배포 전 정리와 커밋 준비 | Pages 경로, 공개 가능성 확인 | 높음 | 배포 승인 | 배포 취소/이전 버전 복구 | 부분 준비 | 승인 전 최종 점검 필요 |
| 7. 배포 후 | MANUAL | 없음 | 공개 URL 상태 확인 | 중간 | 운영 권한 | 이전 배포로 롤백 | 미준비 | 모니터링과 상태 확인 수동 수행 |

## 지금 자동화하기 가장 좋은 루프
- `루프 1. 정적 뼈대`
- 이유: 입력이 단순하고, 실패 원인이 구조·링크·문법으로 제한되며, `claude-sonnet-5` 정적 검증이 이미 PASS였다.

## 사람이 반드시 승인해야 하는 지점
- 프로필의 사실 문구와 최종 톤
- Snake 게임의 조작 방식과 난이도
- GitHub Pages 배포 시작과 공개 전환
- 코드가 아닌 콘텐츠 삭제나 큰 구조 변경

## Claude Verifier가 확인해야 할 항목
- 파일 존재와 상대 경로
- HTML 구조와 내부 링크
- `meta viewport`
- CSS 반응형 구조와 브레이크포인트
- JavaScript 문법과 명백한 런타임 위험
- Games 섹션 준비 상태와 게임 입력/충돌
- GitHub Pages 정적 호환성

## Claude 사용 불가 시 Fallback
- `CODEX_FALLBACK`
- Codex가 수정과 테스트를 모두 수행
- 실패 원인과 fingerprint, 사용 불가 이유를 `AORR_LOG.md`와 `MEMORY.md`에 남김

## 운영 전 필요한 테스트·권한·모니터링
- 테스트: 정적 verifier, 브라우저 런타임, 375px / 768px / 1440px, 모바일 터치, GitHub Pages 공개 URL
- 권한: 파일 편집, 필요 시 unsandboxed Claude CLI, 배포 승인
- 모니터링: 콘솔 오류, 깨진 링크, 반응형 깨짐, 페이지 응답 코드, 게임 입력 지연

## 현업 적용 다음 액션
1. `Snake Game` 실제 입력/충돌 루프를 구현하고 Claude Verifier로 재검증한다.
2. 프로필 카피를 최종 승인받아 `About` / `Projects` / `Experience`를 마감한다.
3. 배포 전 회귀 체크리스트를 확정하고 GitHub Pages 전환 전 승인 절차를 고정한다.
